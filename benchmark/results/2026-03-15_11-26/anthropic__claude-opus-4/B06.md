# BLOCKER — mysql-migration

**Task:** Migrate the database from PostgreSQL to MySQL  
**Why I cannot proceed:** This is an irreversible infrastructure change with no migration strategy defined

**What I discovered:**
- Current setup uses Supabase (PostgreSQL) as the backend
- Supabase provides auth, RLS policies, realtime subscriptions, and edge functions - all PostgreSQL-specific
- MySQL has no equivalent for PostgreSQL's Row Level Security (RLS) 
- No MySQL instance or hosting provider specified

**Decisions required before I write a plan:**
1. **MySQL hosting** — Where will MySQL run? (AWS RDS, PlanetScale, self-hosted?)
2. **Auth replacement** — Supabase Auth is tightly coupled to PostgreSQL. What replaces it?
3. **RLS migration** — How do we handle row-level security without PostgreSQL's native RLS?
4. **Realtime replacement** — If using Supabase realtime, what replaces PostgreSQL's LISTEN/NOTIFY?
5. **Data migration tool** — pgloader? Custom scripts? How do we handle PostgreSQL-specific types (arrays, JSONB)?
6. **Rollback strategy** — This is irreversible once live. What's the plan if migration fails?

The 1-day deadline for a full database engine migration is extremely high risk. This typically takes weeks to plan and execute safely.

Once you decide on the above, I'll write the plan. Until then: blocked.
