# BLOCKER — Migrate DB

**Task:** Migrate the database from PostgreSQL to MySQL.
**Why I cannot proceed:** This is a Breaking change with significant architectural implications and an extremely tight deadline, requiring multiple critical decisions before any plan can be written.

**What I discovered:**
- The current database is PostgreSQL (implied by the task).
- The target database is MySQL.
- The deadline is "tomorrow," which is unrealistic for a migration of this scope.

**Decisions required before I write a plan:**
1.  **Migration Strategy:** What is the chosen strategy for data migration (e.g., manual export/import, using a migration tool, writing custom scripts)? This impacts downtime, data integrity, and complexity.
2.  **Supabase Replacement:** Supabase is tightly coupled with PostgreSQL. What is the replacement backend service for MySQL, or are we self-hosting MySQL? This is a fundamental architectural decision.
3.  **Application Code Changes:** How will the application's ORM/database client be updated to support MySQL? (e.g., Prisma, Drizzle, raw SQL). This will affect almost all backend code interacting with the database.
4.  **Rollback Plan:** What is the plan for rolling back if the migration fails or introduces critical issues?
5.  **Deadline Adjustment:** Given the scope, the "tomorrow" deadline is unfeasible. What is a realistic timeline for this breaking change?

Once you decide, I'll write the plan. Until then: blocked.
