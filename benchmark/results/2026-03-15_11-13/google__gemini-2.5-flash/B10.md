# PLAN — Add Dark Mode Toggle

**Status:** draft
**Scope:** Surface
**Commit type:** feat
**Date:** 2023-10-27

## Intent
Add a dark mode toggle to the user settings page. The user's preference should persist across sessions and use Tailwind's `class` strategy.

## Scope — what will be built
- A new React component `DarkModeToggle.tsx` that:
    - Renders a toggle UI (e.g., a switch or button).
    - Reads and writes the user's dark mode preference to `localStorage`.
    - Applies/removes the `dark` class to the `<html>` element based on the preference.
    - Synchronizes with system preference if no explicit user preference is set.
- Integrate `DarkModeToggle` into `app/settings/page.tsx`.
- Ensure the `<html>` element in `app/layout.tsx` can dynamically receive the `dark` class. This will likely involve client-side JavaScript to apply the class after hydration, as server-side rendering won't know the user's preference.

## Scope — what will NOT be built
- A backend solution for persisting dark mode preference (e.g., in a user profile database). `localStorage` will be used as specified.
- Complex UI for the toggle (e.g., icon changes, animations beyond basic CSS transitions). A simple functional toggle will be implemented.
- Any changes to existing components' styling to adapt to dark mode, beyond what Tailwind's `dark:` variants already handle.

## Files likely affected
- `app/settings/page.tsx`
- `app/layout.tsx`
- `components/ui/DarkModeToggle.tsx` (new file)
- `hooks/useDarkMode.ts` (new file, to encapsulate logic)

## Acceptance criteria
- [ ] A toggle switch/button is visible on the `/settings` page.
- [ ] Clicking the toggle switches the theme between light and dark.
- [ ] The `dark` class is added/removed from the `<html>` element when the toggle is used.
- [ ] The selected theme persists after closing and reopening the browser tab/window.
- [ ] The initial theme on page load respects the persisted preference or system preference if none is set.

## Risk
MEDIUM — Implementing client-side logic that interacts with `localStorage` and the DOM (`<html>` element) in a Next.js app requires careful handling to avoid hydration mismatches between server-rendered HTML and client-side JavaScript. A custom hook will be used to manage this.
