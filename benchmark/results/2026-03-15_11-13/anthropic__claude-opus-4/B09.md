Since you expressed uncertainty about WebSockets vs SSE, I'll shadow branch both options and compare with real code - my questions are only about requirements.

**Interview questions:**

1. **Notification types and volume**: What types of notifications will you send (user actions, system alerts, chat messages)? Expected frequency per user?

2. **Directionality**: Do you need bidirectional communication (client can send data back through the same connection) or just server-to-client push?

3. **Infrastructure constraints**: Where are you deploying (Vercel, AWS, self-hosted)? Some platforms have WebSocket limitations.

4. **Connection persistence**: Do users need notifications while browsing different pages, or only on specific screens?

5. **Scale requirements**: Rough estimate of concurrent users? This affects whether we need connection pooling or can use simpler approaches.
