# BLOCKER — Migrate DB

**Task:** Migrate the database from PostgreSQL to MySQL.
**Why I cannot proceed:** This is a Breaking change with significant architectural implications and an extremely tight deadline, requiring multiple critical decisions before any plan can be written.

**What I discovered:**
- The current database is PostgreSQL (implied by the task description).
- The target database is MySQL.
- The deadline is "tomorrow," which is unrealistic for a migration of this scope.

**Decisions required before I write a plan:**
1. What is the migration strategy for data? (e.g., dump/restore, ETL tool, manual script, third-party service). This impacts downtime, data integrity, and complexity.
2. How will the application's ORM/database client be updated to support MySQL? (e.g., Prisma, Drizzle, raw SQL changes). This is a systemic change across the entire application.
3. What is the rollback plan in case of failure? A database migration is high-risk.
4. What is the plan for Supabase features that are PostgreSQL-specific (e.g., RLS, Edge Functions, Auth)? Supabase is a PostgreSQL-based platform; migrating to MySQL means abandoning Supabase for the database layer, which has massive implications for the entire backend architecture.

Once you decide, I'll write the plan. Until then: blocked.
